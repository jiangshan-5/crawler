# 通用网页爬虫工具使用指南 🕷️

## 简介

通用网页爬虫工具 v2.0 是一个功能强大的网页数据爬取工具，支持：

- ✅ 自定义目标 URL
- ✅ 自定义 CSS 选择器
- ✅ 控制爬取数量
- ✅ 自定义保存位置
- ✅ 支持 JSON 和 CSV 格式
- ✅ 实时日志和数据预览
- ✅ 列表页面和单页面两种模式

## 快速开始 🚀

### 启动工具

**方法 1：双击启动**
```
双击 run_universal_crawler.bat
```

**方法 2：命令行启动**
```bash
cd crawler
python src/universal_crawler_gui.py
```

## 界面说明 📱

### 左侧：配置区域

#### 1. URL 配置
- **目标 URL**：输入要爬取的网页地址
- **爬取模式**：
  - 列表页面：爬取包含多个项目的列表页
  - 单个页面：爬取单个页面的数据

#### 2. 字段选择器配置
- 定义要提取的数据字段
- 每个字段包含：
  - 字段名：数据的名称（如：标题、价格）
  - CSS 选择器：用于定位元素的选择器
  - 说明：字段的描述（可选）

#### 3. 爬取控制
- **爬取数量**：限制爬取的数据条数
- **请求延迟**：每次请求之间的延迟时间（秒）
- **保存格式**：JSON 或 CSV
- **保存位置**：数据保存的目录

### 右侧：日志和预览

#### 1. 运行日志
- 实时显示爬取进度
- 彩色日志（INFO/SUCCESS/WARNING/ERROR）
- 自动滚动

#### 2. 数据预览
- 显示前 5 条爬取的数据
- JSON 格式预览
- 方便验证数据正确性

## CSS 选择器语法 📝

### 基本语法

| 选择器 | 说明 | 示例 |
|--------|------|------|
| `tag` | 标签选择器 | `h1`, `p`, `div` |
| `.class` | 类选择器 | `.title`, `.price` |
| `#id` | ID 选择器 | `#main`, `#header` |
| `tag.class` | 组合选择器 | `div.item`, `span.price` |
| `tag@attr` | 提取属性 | `a@href`, `img@src` |

### 常用选择器示例

```css
/* 提取文本 */
h1                    /* 提取 h1 标签的文本 */
div.title             /* 提取 class 为 title 的 div 文本 */
span.price            /* 提取 class 为 price 的 span 文本 */

/* 提取属性 */
a@href                /* 提取 a 标签的 href 属性 */
img@src               /* 提取 img 标签的 src 属性 */
div@data-id           /* 提取 div 的 data-id 属性 */

/* 复杂选择器 */
div.item > h2         /* 提取 div.item 的直接子元素 h2 */
ul li:first-child     /* 提取 ul 的第一个 li */
div.content p         /* 提取 div.content 内的 p 标签 */
```

## 使用示例 💡

### 示例 1：爬取新闻列表

**目标**：爬取新闻网站的标题、链接和时间

**配置**：
1. URL：`https://news.example.com`
2. 模式：列表页面
3. 列表容器：`div.news-item`
4. 字段配置：
   - 标题：`h2.title`
   - 链接：`a@href`
   - 时间：`span.time`
5. 爬取数量：20
6. 保存格式：JSON

**配置**：
1. URL：`https://shop.example.com/products`
2. 模式：列表页面
3. 列表容器：`div.product`
4. 字段配置：
   - 商品名：`h3.name`
   - 价格：`span.price`
   - 图片：`img@src`
   - 评分：`span.rating`
5. 爬取数量：50
6. 保存格式：CSV

### 示例 3：爬取单个页面

**目标**：爬取文章页面的标题、作者、内容

**配置**：
1. URL：`https://blog.example.com/article/123`
2. 模式：单个页面
3. 字段配置：
   - 标题：`h1.article-title`
   - 作者：`span.author`
   - 时间：`time@datetime`
   - 内容：`div.content`
4. 保存格式：JSON

## 操作流程 📋

### 完整流程

1. **启动工具**
   - 双击 `run_universal_crawler.bat`

2. **配置 URL**
   - 输入目标网页地址
   - 选择爬取模式（列表/单页）
   - 如果是列表模式，输入列表容器选择器

3. **配置字段**
   - 点击"➕ 添加字段"
   - 输入字段名和 CSS 选择器
   - 重复添加所有需要的字段
   - 或点击"📋 示例"加载示例配置

4. **设置控制参数**
   - 设置爬取数量
   - 设置请求延迟
   - 选择保存格式
   - 选择保存位置

5. **开始爬取**
   - 点击"🚀 开始爬取"
   - 观察日志输出
   - 查看数据预览

6. **查看结果**
   - 爬取完成后会弹出提示
   - 在保存位置查看数据文件

## 如何找到正确的选择器 🔍

### 方法 1：使用浏览器开发者工具

1. 打开目标网页
2. 按 F12 打开开发者工具
3. 点击"选择元素"按钮（或按 Ctrl+Shift+C）
4. 点击要爬取的元素
5. 在开发者工具中查看元素的标签、类名、ID
6. 构建 CSS 选择器

### 方法 2：使用 Chrome 的 Copy Selector

1. 打开目标网页
2. 按 F12 打开开发者工具
3. 右键点击要爬取的元素
4. 选择"Copy" → "Copy selector"
5. 粘贴到工具中（可能需要简化）

### 方法 3：测试选择器

在浏览器控制台中测试：
```javascript
// 测试选择器是否正确
document.querySelector('div.item')        // 单个元素
document.querySelectorAll('div.item')     // 多个元素
```

## 常见问题 ❓

### Q1: 没有提取到数据？
**A:** 检查以下几点：
1. URL 是否正确
2. 选择器是否正确（使用浏览器开发者工具验证）
3. 网页是否需要登录
4. 网页是否使用 JavaScript 动态加载（需要使用 Selenium）

### Q2: 提取的数据不完整？
**A:** 可能原因：
1. 选择器不够精确，匹配到了错误的元素
2. 网页结构复杂，需要更具体的选择器
3. 数据在多个元素中，需要调整选择器

### Q3: 爬取速度太慢？
**A:** 
1. 减少请求延迟（但要注意不要太快）
2. 减少爬取数量
3. 检查网络连接

### Q4: 如何爬取需要登录的网站？
**A:** 
1. 在浏览器中登录
2. 复制 Cookie
3. 在代码中添加 Cookie（需要修改源代码）

### Q5: 如何爬取动态加载的内容？
**A:** 
- 当前版本不支持 JavaScript 渲染
- 需要使用 Selenium 版本（未来版本）
- 或者找到 API 接口直接爬取

## 高级技巧 🎓

### 1. 提取嵌套数据

```css
/* 提取嵌套在多层 div 中的数据 */
div.container div.item h2.title
```

### 2. 提取多个属性

添加多个字段，每个字段提取不同属性：
- 链接：`a@href`
- 标题：`a@title`
- 文本：`a`

### 3. 使用伪类选择器

```css
/* 第一个元素 */
li:first-child

/* 最后一个元素 */
li:last-child

/* 第 n 个元素 */
li:nth-child(2)
```

### 4. 组合多个选择器

```css
/* 多个类 */
div.item.active

/* 后代选择器 */
div.container span.price

/* 子选择器 */
div.item > h2
```

## 注意事项 ⚠️

### 法律和道德

1. **遵守 robots.txt**：尊重网站的爬虫协议
2. **控制频率**：不要对网站造成过大压力
3. **合法使用**：仅用于学习和个人用途
4. **版权**：尊重数据版权，不要用于商业用途

### 技术限制

1. **静态页面**：仅支持静态 HTML 页面
2. **JavaScript**：不支持 JavaScript 渲染的内容
3. **验证码**：无法处理验证码
4. **登录**：需要手动处理登录状态
