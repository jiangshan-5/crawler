# 通用爬虫工具 - 升级说明 🚀

## 升级概述

基于原有的图标爬虫工具，我们开发了全新的通用网页爬虫工具 v2.0。

## 新增功能 ✨

### 1. 自定义 URL
- ✅ 用户可以粘贴任意网址
- ✅ 不再局限于特定的图标网站
- ✅ 支持任何静态 HTML 网页

### 2. 自定义数据提取
- ✅ 用户可以指定要爬取的数据字段
- ✅ 使用 CSS 选择器定位元素
- ✅ 支持提取文本和属性
- ✅ 可视化配置界面

### 3. 完全控制
- ✅ 控制爬取数量（1-1000 条）
- ✅ 控制请求延迟（0-10 秒）
- ✅ 选择保存格式（JSON/CSV）
- ✅ 自定义保存位置

### 4. 实时反馈
- ✅ 实时日志显示
- ✅ 数据预览功能
- ✅ 爬取统计信息
- ✅ 彩色日志分级

### 5. 两种模式
- ✅ 列表页面模式：爬取多个项目
- ✅ 单页面模式：爬取单个页面

## 与原工具的对比 📊

| 功能 | 图标爬虫 | 通用爬虫 |
|------|----------|----------|
| 目标网站 | 固定 5 个图标网站 | 任意网站 ✅ |
| 数据类型 | 图标（SVG/PNG） | 任意数据 ✅ |
| 选择器 | 预设 | 用户自定义 ✅ |
| 爬取数量 | 固定或有限 | 完全自定义 ✅ |
| 保存格式 | JSON | JSON + CSV ✅ |
| 保存位置 | 固定目录 | 用户自定义 ✅ |
| 数据预览 | 无 | 实时预览 ✅ |
| 使用场景 | 图标爬取 | 通用数据爬取 ✅ |

## 快速上手 🎯

### 步骤 1：启动工具
```bash
双击 run_universal_crawler.bat
```

### 步骤 2：配置 URL
```
输入目标网址：https://example.com
选择模式：列表页面
列表容器：div.item
```

### 步骤 3：添加字段
```
字段 1：
  字段名：标题
  选择器：h2.title

字段 2：
  字段名：链接
  选择器：a@href

字段 3：
  字段名：时间
  选择器：span.time
```

### 步骤 4：设置控制
```
爬取数量：20
请求延迟：1 秒
保存格式：JSON
保存位置：data/crawled_data
```

### 步骤 5：开始爬取
```
点击"🚀 开始爬取"
观察日志和预览
等待完成
```

## CSS 选择器速查 📝

### 基本选择器
```css
h1                  /* 标签 */
.class              /* 类 */
#id                 /* ID */
div.class           /* 组合 */
```

### 提取属性
```css
a@href              /* 链接 */
img@src             /* 图片 */
div@data-id         /* 数据属性 */
```

### 层级选择器
```css
div > p             /* 子元素 */
div p               /* 后代元素 */
div + p             /* 相邻元素 */
```

### 伪类选择器
```css
li:first-child      /* 第一个 */
li:last-child       /* 最后一个 */
li:nth-child(2)     /* 第 n 个 */
```

## 常见问题 ❓

### Q: 与图标爬虫有什么区别？
A: 图标爬虫专门用于爬取图标，通用爬虫可以爬取任意网页数据。

### Q: 可以同时使用两个工具吗？
A: 可以，它们是独立的工具，互不影响。

### Q: 通用爬虫可以爬取图标吗？
A: 可以，但需要手动配置选择器，不如图标爬虫方便。

### Q: 如何学习 CSS 选择器？
A: 查看 [通用爬虫指南](./UNIVERSAL_CRAWLER_GUIDE.md) 中的详细说明。

### Q: 支持动态加载的网页吗？
A: 当前版本不支持，需要使用 Selenium（未来版本）。
